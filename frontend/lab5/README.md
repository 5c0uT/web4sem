# lab5

Редактор воксельной модели с тремя 2D-проекциями, 3D-просмотром на Three.js и сохранением через `json-server`.

## Запуск

### Вариант 1. Из папки `frontend`

```bash
cd frontend
npm run server:lab5
```

Этот скрипт появился в корневом `frontend/package.json` и просто проксирует запуск в `lab5`.

### Вариант 2. Напрямую из папки `lab5`

```bash
cd frontend/lab5
npm install
npm run server
```

Дополнительно оставлен алиас:

```bash
npm run server:lab5
```

## Почему раньше команда не работала

Потому что в `frontend` не было `package.json`, а значит `npm` не видел ни одного скрипта.
Теперь в корне `frontend` добавлен минимальный `package.json` с командами:

- `server:lab5`
- `lint:lab5`
- `install:lab5`

## Как открыть клиентскую часть

Во втором терминале откройте папку `lab5` через локальный статический сервер.

Варианты:

- VS Code `Live Server`;
- `npx serve .`;
- любой другой локальный HTTP-сервер.

Если используется `Live Server`, достаточно открыть `index.html` из папки `lab5`.

## Общие модули

- `../vendor` содержит общие модули Three.js;
- `../algorithm/voxelProjection.js` содержит общую логику работы с проекциями и вокселями;
- `../common` содержит общее состояние редактора, рендереры и базовые стили.

## Что хранится в `db.json`

Ресурс `models` содержит:

- `id` — идентификатор записи;
- `name` — имя модели;
- `createdAt` — дата создания;
- `updatedAt` — дата последнего изменения;
- `snapshot` — сохраненное состояние редактора.

### Новый формат проекций

Проекции теперь хранятся не как массивы `true` / `false`, а как многострочные строки из `0` и `1`.

Пример для сетки `2 x 2`:

```text
{
  "top": "00\n11",
  "front": "10\n10",
  "left": "11\n00"
}
```

Такой формат:

- заметно компактнее;
- легко читается глазами;
- удобен для объяснения на защите.

Редактор при загрузке понимает и новый компактный формат, и старый формат со значениями `true` / `false`.
