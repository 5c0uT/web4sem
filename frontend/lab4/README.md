# Лабораторная работа 4 - 3D-редактор

Цель:

- разработать веб-приложение для создания низкополигональных 3D-моделей;
- реализовать построение формы по трем 2D-проекциям;
- применить алгоритм Space Carving для пересчета воксельной модели;
- выполнить отбраковку внутренних граней (Voxel Face Culling);
- добавить режим покраски модели и 3D-просмотр на Three.js.

Функции:

- четыре панели одинакового назначения: вид сверху, вид спереди, вид слева и 3D-просмотр;
- фоновая сетка на всех 2D-canvas;
- синхронизированное перекрестие между тремя проекциями;
- режим «Форма»:
  - ЛКМ добавляет пиксель на проекции;
  - ПКМ удаляет пиксель на проекции;
  - после изменения силуэта выполняется пересчет 3D-объема по Space Carving;
- режим «Покраска»:
  - форма модели блокируется;
  - можно выбрать цвет через color picker;
  - покраска доступна через 2D-проекции (на первый видимый воксель по оси);
  - покраска доступна кликом по 3D-модели через Raycasting;
- 3D-рендеринг на Three.js;
- OrbitControls для вращения, приближения и отдаления модели;
- генерация только внешних граней модели через Face Culling.

Структура кода:

- `index.html` — разметка страницы и подключение модулей;
- `styles.css` — оформление интерфейса;
- `app.js` — логика приложения;
  - состояние приложения;
  - работа с проекциями;
  - математика вокселей;
  - 3D-рендеринг;
  - обработка UI;
- `../vendor/three.module.js` и `../vendor/OrbitControls.js` — общие модули Three.js;
- `../algorithm/voxelProjection.js` — общая логика работы с проекциями и вокселями.

Запуск:

- открыть проект через локальный сервер, например Live Server / Go Live;
- убедиться, что в корне `frontend` существует папка `vendor` с модулями Three.js;
- открыть страницу в браузере.

Примечания:

- в режиме формы цвет существующих вокселей сохраняется при пересчете, если воксель остается в модели;
- внутренние грани не создаются, поэтому модель отображается легче и чище;
- HTML не создается строками внутри JavaScript, разметка полностью находится в `index.html`.
